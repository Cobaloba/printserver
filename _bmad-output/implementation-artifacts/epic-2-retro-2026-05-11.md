## Challenges

- **Both demos failed initially** — `pyusb` missing from Docker image (story 2.1 gap undetected until demo), then `privileged: true` missing (USB kernel driver detach failing silently). Both required code changes before printing worked.
- **SSH username confusion** — Pi username is `conor`, not `pi`; had `pi@192.168.0.25` in memory and wasted a debug loop before discovering the error
- **Silent failures** — `except Exception: pass` in lifespan hid non-printer errors; `str(e) == ""` on USB exceptions produced blank 503 responses; both actively obscured root causes
- **Validator gaps** — 9 new tests added in code review for validators that should have been written during development (`ReceiptRequest.items` empty, `tax_pct` negative, `font_size` literal, `NewRollRequest` bounds)

---

## Key Lessons

1. **Demos are integration tests** — mock tests cannot catch the hardware/container boundary. Real hardware demos must be part of the definition of done for any story touching the print pipeline, Dockerfile, docker-compose, or a printer-wired endpoint.

2. **Silent failures are the worst failures** — `except Exception: pass` and `str(e) == ""` both hid real production problems for multiple stories. Narrow exception handling and explicit error messages are non-negotiable.

3. **Code review catches what tests miss** — 21 patches from 4 structured review groups; schedule code review as a formal story-cycle step, not an afterthought.
