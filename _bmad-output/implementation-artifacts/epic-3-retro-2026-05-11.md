## Challenges

- **QR code crashed USB connection** — native `p.qr()` partially executes on the Jolimark before raising, feeding paper and corrupting USB state (Errno 5). The `try/except` fallback then sent a large raster image on top of an already-broken connection. Required printer power cycle to recover. Fixed by skipping native command entirely and always using raster fallback with `box_size=6`
- **Font size selector silently did nothing** — `p.set(height=2, width=2)` in python-escpos requires `custom_size=True` to take effect. Without it, the parameters are ignored. Compiled fine, MockPrinter tests passed, real hardware exposed it immediately. Fixed by switching to font B / font A / custom_size pattern
- **USB singleton has no reconnect path** — printer power cycle requires container restart. Workaround: admin page "Restart Service" button. Root cause deferred
- **UX gaps only visible on mobile** — back navigation, restart button, QR error message not clearing — none in ACs, all obvious within minutes of real phone use
- **`python-escpos` API surprises** — library behaviour doesn't always match docs or reasonable assumptions. `set()` silently ignores parameters. `qr()` corrupts state on unsupported hardware

---

## Key Lessons

1. **Hardware behaviour cannot be fully spec'd in advance — live testing is non-negotiable**
   Both hardware bugs (QR crash, font size) were invisible to MockPrinter and the build. They only appeared when data hit the physical Jolimark. This reinforces Epic 2's lesson: hardware integration testing is the only complete test.

2. **UX gaps surface immediately on mobile — design for no-chrome browsers**
   Back navigation, the restart button, QR error persistence — none were in the ACs, all were obvious within minutes of phone use. Stories touching UI should include "use it on a phone" as an implicit DoD check.

3. **New ESC/POS commands need live print tests — never trust MockPrinter alone**
   Whenever a new ESC/POS command is added, assume it needs a physical print test. Library assumptions (silent no-ops, partial execution before errors) can only be discovered on real hardware.

4. **Small timing decisions have outsized UX impact**
   Five seconds of "Offline" on every page load felt broken even though it was technically correct. The immediate-first-poll fix was one line but meaningfully improved perceived quality.
